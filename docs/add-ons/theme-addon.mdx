# Overview

The theme Add-on provides theming for previewing your use-cases. 
Therefore, your components are previewed in the same design language than your app.

# Material Theme Add-on

The Material Theme Add-on provides your Material theme based on the `ThemeData` class. 

<Tabs
  groupId="approach"
  values={[
    { label: 'Generator', value: 'generator' },
    { label: 'Manual', value: 'manual' },
  ]}
>
  <TabItem value="generator">
    To use the `MaterialThemeAddon` with the [widgetbook_generator package](https://pub.dev/packages/widgetbook_generator) annotate a function that returns your `ThemeData` with the `@WidgetbookTheme` annotation.

    ```dart title="Example: @WidgetbookTheme for MaterialThemeAddon"
    ThemeData getTheme() => ThemeData.dark();
    ```

    <Info>
        You can use the `@WidgetbookTheme` annotation on multiple theme instances.
    </Info>
  </TabItem>
  <TabItem value="manual">
    The `MaterialThemeAddon` requires a `MaterialThemeSetting` that has two properties.
    The `themes` property defines which themes are available while the `activeTheme` property defines which theme is active and selected.

    ```dart title="Example: MaterialThemeAddon"
    final themeAddon = MaterialThemeAddon(
      setting: MaterialThemeSetting(
        themes: [
          WidgetbookTheme(
            name: 'Light',
            data: lightTheme(),
          ),
          WidgetbookTheme(
            name: 'Dark',
            data: darkTheme(),
          ),
        ],
        activeTheme: WidgetbookTheme(
          name: 'Light',
          data: lightTheme(),
        ),
      ),
    );
    ```
  </TabItem>
</Tabs>

# Custom Theme Add-on

In addition to Material and Cupertino themes, Widgetbook allows to define completely custom themes as well. 

Let us assume you have defined a very simple custom theme `AppThemeData` that just holds a single color.

<Info>
    The `AppThemeData` class can be compared with the `ThemeData` class for the Material design system or the `CupertinoThemeData` class for the Cupertino design system.
</Info>

```dart title="Example: AppThemeData"
class AppThemeData {
  final Color color;

  AppThemeData({
    required this.color,
  });
}
```

Now you need to inject this `AppThemeData` object into the Widget tree.
This is commonly done by using an `InheritedWidget`.
Therefore, let us assume you define a `AppTheme` `InheritedWidget` that injects your custom theme into the Widget tree. 

<Info>
    The `AppTheme` widget can be compared with the `Theme` Widget for the Material design system or the `CupertinoTheme` Widget for the Cupertino design system.
</Info>

```dart title="Example: AppTheme InheritedWidget"
class AppTheme extends InheritedWidget {
  const AppTheme({
    Key? key,
    required this.data,
    required Widget child,
  }) : super(
          key: key,
          child: child,
        );

  final AppThemeData data;

  static AppThemeData of(BuildContext context) {
    final widget = context.dependOnInheritedWidgetOfExactType<AppTheme>();
    return widget!.data;
  }

  @override
  bool updateShouldNotify(covariant AppTheme oldWidget) {
    return data != oldWidget.data;
  }
}
```

In addition to the class definitions, we define two custom theme instances

```dart title="Example: Custom themes"
final blueTheme = AppThemeData(
  color: Colors.blue,
);

final yellowTheme = AppThemeData(
  color: Colors.yellow,
);
```

The Custom Theme Add-on provides your custom theme based on the `AppThemeData` class.

  <TabItem value="generator">
    To use the `CustomThemeAddon` with the [widgetbook_generator package](https://pub.dev/packages/widgetbook_generator) annotate a function that returns your `AppThemeData` with the `@WidgetbookTheme` annotation.

    ```dart title="Example: @WidgetbookTheme for custom theme"

    ```

    <Info>
        You can use the `@WidgetbookTheme` annotation on multiple theme instances.
    </Info>
  </TabItem>
  <TabItem value="manual">
    The `CustomThemeAddon` requires a `CustomThemeSetting` that has a generic parameter and two properties.
    The `themes` property defines which themes are available while the `activeTheme` property defines which theme is active and selected.

    ```dart title="Example: CustomThemeAddon"
    final themeAddon = CustomThemeAddon<AppThemeData>(
      setting: CustomThemeSetting(
        themes: [
          WidgetbookTheme(
            name: 'Blue',
            data: blueTheme,
          ),
          WidgetbookTheme(
            name: 'Yellow',
            data: yellow,
          ),
        ],
        activeTheme: WidgetbookTheme(
            name: 'Blue',
            data: blueTheme,
        ),
      ),
    );
    ```
  </TabItem>
</Tabs>

<Warning>For the theme to be properly injected into Widgetbook's Widget tree, make sure to implement a custom [`appBuilder` method](/getting-started/configuration#customize-how-use-cases-are-previewed) function.</Warning>
